# 이동 애니메이션 버그 해결

**날짜**: 2026년 4월 18일  
**게임**: 2D 플랫포머 (탑뷰)

---

## 🔴 문제 상황

Move 상태임에도 불구하고 **Idle 애니메이션이 재생되는 현상**
- Idle → Move로 전환할 때 Move 상태인데 Idle 애니메이션이 나옴

---

## 🔍 시도했던 방법

### Has Exit Time 해제 + Transition Duration 0으로 변경
**생각**: 애니메이션 전환 설정 문제인가?

**해본 것**:
- Animator에서 **Has Exit Time** 체크 해제
- **Transition Duration** 값을 0으로 변경

**결과**: ❌ 해결 안됨

---

## ✅ 해결 방법

### 진짜 원인

Trigger 방식의 문제였음

1. 게임 시작 시 기본 상태가 Idle이고, Idle 상태를 **Trigger**로 실행시킴
2. 근데 이미 Idle 상태여서 Trigger가 실행되지 않고 **대기 상태**에 놓임
3. Move 상태를 받아도 대기 중이던 **Idle Trigger가 먼저 실행**됨
4. 다시 멈춰도 Idle Trigger가 Start되면서 또 대기 상태에 놓임
5. 이게 반복됨

### 해결책 3가지

**1번째: Trigger 강제로 비우기**
- Trigger 실행 전에 아래 코드로 강제로 비워줌
```csharp
Animator.ResetTrigger("Idle");
```

**2번째: 최초 1회 Idle Trigger 실행 안하기**
- 게임 첫 실행 시에는 Idle Trigger를 발생시키지 않음

**3번째: Trigger 발생 시점 변경**
- Idle 상태가 **Start될 때** Trigger 발생 ❌
- Move가 **Exit될 때** Idle Trigger 발생 ✅

---

## 💡 배운 점

- Trigger는 사용되지 않으면 대기 상태로 남아있음
- 이미 해당 상태일 때 같은 상태의 Trigger를 실행하면 소비되지 않고 쌓임
- Trigger 방식 사용 시 **ResetTrigger**로 관리하거나 발생 시점을 잘 설계해야 함
